Overview

The AgentPipe Web design system provides a cohesive visual language for building a privacy-first, realtime multi-agent AI orchestration dashboard. It emphasizes clarity, consistency, and accessibility while supporting 14 different AI agents with unique visual identities.

Design Principles

  • Clarity Over Cleverness: Prioritize intuitive, self-explanatory interfaces
  • Progressive Disclosure: Show essential information first, reveal complexity gradually
  • Feedback & Affordance: Every interaction provides clear visual feedback
  • Consistency: Maintain predictable patterns across the entire application
  • Performance: Design feels instant with optimistic UI updates and skeleton screens
  • Accessibility First: Design for all users, including those with disabilities

Technology Stack

  • Framework: Next.js 15 with App Router and Server Components
  • Styling: Tailwind CSS v4 (CSS-based configuration, no JS config)
  • Components: shadcn/ui adapted for Tailwind v4
  • Type Safety: TypeScript with strict mode

AgentDetails

A side panel component showing comprehensive agent information including description, statistics, available models, and external links. Displays in a sticky sidebar on desktop and as a full-screen overlay on mobile.

Layout Structure

  • Header: Large avatar (lg size), agent name, official name, close button (mobile)
  • Content (scrollable):
    • Tagline (italic, primary color)
    • Description paragraph
    • Provider and Category metadata
    • Available models list
    • Statistics panel with grid layout
    • External links (website, GitHub, documentation)

Responsive Behavior

Breakpoint Behavior
Mobile (< 1024px) Full-screen overlay with close button, shown when agent selected
Desktop ( 1024px) Sticky sidebar, always visible, close button hidden

Empty State

When no agent is selected, displays a centered icon with "Select an agent to view details" message (desktop only).

Code

<AgentDetails
  agent={selectedAgent}
  onClose={() => setSelectedAgent(null)}
/>

AgentsPageClient

Master-detail layout component for the agents directory page. Displays agent cards in categorized grids with a detail panel that responds to selection.

Layout Structure

  • Header: Page title and description
  • Summary Stats: 4-column grid (2 cols on mobile) showing totals
  • Main Layout: 3-column grid (lg breakpoint)
    • Left 2 columns: Agent cards grouped by category
    • Right 1 column: AgentDetails sticky sidebar

Category Grouping

Agents are automatically grouped into two categories:

  • Commercial Agents: Paid services (Claude, Gemini, Copilot, Cursor, etc.)
  • Open Source Agents: Self-hosted options (Ollama, OpenCode, Codex, etc.)

Accessibility Guidelines

AgentPipe Web is designed to meet WCAG 2.1 Level AA standards. All components are built with accessibility as a core requirement, not an afterthought.

Color Contrast

All text and interactive elements meet or exceed WCAG 2.1 AA contrast requirements.

Element Type Minimum Ratio Our Standard
Normal text (< 18px) 4.5:1 4.5:1 or higher
Large text ( 18px or 14px bold) 3:1 3:1 or higher
Interactive elements (borders, icons) 3:1 3:1 or higher
Agent colors (all 14) 4.5:1 Verified 4.7:1 to 6.1:1

Keyboard Navigation

All interactive elements are keyboard accessible with clear focus indicators.

Focus Indicators

Clear, visible focus indicators help keyboard users navigate the interface. 

<!-- Default focus ring (2px outline, 2px offset) -->
.focus-ring:focus-visible {
  outline: 2px solid var(--ring);
  outline-offset: 2px;
}

<!-- Usage in components -->
<button class="focus-ring">Button</button>
<a href="#" class="focus-ring">Link</a>

ARIA Labels & Roles

Proper ARIA attributes ensure screen readers can convey meaning and context.

Attribute Usage Example
aria-label Label for elements without visible text <button aria-label="Close panel"></button>
role Semantic role for custom elements <div role="button">...</div>
aria-current Indicate current page in navigation <a aria-current="page">Agents</a>
aria-expanded State of collapsible elements <button aria-expanded="true">...</button>
aria-live Announce dynamic content changes <div aria-live="polite">Status update</div>

Screen Reader Support

Components are tested with screen readers (NVDA, JAWS, VoiceOver) to ensure proper announcements.

  • Semantic HTML: Use proper heading hierarchy (h1 h2 h3)
  • Skip Links: "Skip to main content" link for keyboard users
  • Live Regions: Announce realtime updates without interrupting flow
  • Alt Text: Descriptive alternative text for all images and icons
  • Form Labels: Every input has an associated label

Reduced Motion

Respect user preferences for reduced motion to prevent vestibular issues. 

<!-- Disable animations for users who prefer reduced motion -->
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}

Accessibility Checklist

Use this checklist when creating new components or features:

  • Color contrast meets 4.5:1 for text, 3:1 for large text/UI elements
  • All interactive elements are keyboard accessible (Tab, Enter, Space)
  • Clear focus indicators visible on all focusable elements
  • Touch targets are at least 44x44px
  • Proper ARIA labels for icon-only buttons and links
  • Semantic HTML with correct heading hierarchy
  • Form inputs have associated labels
  • Error messages are programmatically associated with inputs
  • Loading states announced to screen readers
  • Animations respect prefers-reduced-motion
  • Content is readable at 200% zoom without horizontal scroll
  • Tested with keyboard-only navigation
  • Tested with screen reader (VoiceOver, NVDA, or JAWS)
