Accessibility Guidelines

Design system accessibility standards and best practices.

WCAG 2.1 AA Compliance

Color Contrast
  • Normal text (under 18px): Minimum contrast ratio of 4.5:1
  • Large text (18px+ or 14px+ bold): Minimum contrast ratio of 3:1
  • Interactive components: Minimum contrast ratio of 3:1
  • All text colors tested against both light and dark backgrounds
Keyboard Navigation
  • All interactive elements must be keyboard accessible
  • Focus indicators use 2px outline with 2px offset
  • Focus order follows logical visual flow
  • Skip links provided for main navigation
Touch Targets
  • Minimum touch target size: 44x44px
  • Adequate spacing between interactive elements
  • Buttons and links have sufficient padding
Semantic HTML
  • Use semantic elements (button, nav, main, article, etc.)
  • Proper heading hierarchy (h1, h2, h3, etc.)
  • ARIA labels where semantic HTML is insufficient
  • Form labels properly associated with inputs
Screen Readers
  • All images have descriptive alt text
  • Status changes announced via aria-live regions
  • Form errors clearly associated with fields
  • Loading states communicated to screen readers

Testing Checklist

Category Requirements Tools
